When Running Billing for a Project any tickets with hours in them that are also linked to the Project are not being charged

Last Modified

Mon Oct 22 12:27 GMT 2018

Description

  • When Running Billing for a Project any tickets with hours in them that are also linked to the Project are not being charged

Environment

Solution

  • This is caused by the Project Service Item being configured to bill for the Project Estimate, NOT the actual hours
  • When you create a Project Service Item you can configure the project to bill for the 'Project Estimate' - the estimated dollar amount for the project and the estimated hours to complete it, or you can choose to bill for the 'Actual hours' - tracking all hours in tickets and charging an hourly rate per hour of work
    1. In the 'Customers' tab, select the customer for whom the project service item was created
    2. Click on the 'Service items' link on the left of the page
    3. Click 'Edit' button to the right of the project service item you wish to change
    4. Modify it from Project Estimate to be Actual Hours
    5. Create a new billing batch or delete and re-run your current billing batch to apply changes to invoices for the project
