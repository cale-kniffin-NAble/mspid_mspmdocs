Ticket Mailbox not connecting to Office 365 due to MFA requirement

Last Modified

Fri Jul 17 15:52 GMT 2020

Description

  • As of August, 2019, Microsoft has issued policy guidelines around Conditional Access, or Modern Authentication for its partner program. Currently, this is only policy, and the enforcement guidelines are still fuzzy, with dates in 2020 being suggested.
  • Multi-Factor Authentication requires at least one further layer of authentication in addition to username and password. For example: biometric or physical devices (FIDO-2), verifying a code sent by SMS to a mobile phone, or time-based one time passwords generated in an authenticator app
  • The policy guidelines can be read in the following sections of this document (https://docs.microsoft.com/en-us/microsoft-365/business/set-up-conditional-access-policies):
    1. Require MFA for Admins
    2. End-user protection
    3. Block Legacy authentication
    4. Required MFA for service management.

Environment

  • Ticket Mailboxes
  • Microsoft Office 365 (O365)

Solution

What is the impact on ?
  • uses IMAP to connect to the O365 account, both for the primary MSP, and potentially other MSPs if they are tenants of the same partner account.
  • IMAP is a commonly used used protocol used to retrieve mail from a mailbox, however it only uses single-factor authentication.
  • should ONLY be configured to access O365 mail services using a dedicated, unique, non-admin account. It should never be using an admin or service management account.
  • When is configured this way, a partner can activate MFA on Admin and Service accounts, and gradually rollout MFA to users in an orderly fashion (see End User Protection), without effecting email access.
What will do going forward?
  • Microsoft is recommending using the Graph API as the next generation access to O365 facilities.
  • We will be updating to support this as soon as possible
  • At the time of writing, this is accurate. We are engaged in further discussion with Microsoft to gauge the full impact of this policy.
What should you do for now?
  • We advise you link your Monitoring Mailbox with Office 365 using App Passwords, which is described in this KB article:
    • https://success.solarwindsmsp.com/kb/solarwinds_msp_manager/How-to-link-Monitoring-Mailbox-and-Office-365-with-MFA
