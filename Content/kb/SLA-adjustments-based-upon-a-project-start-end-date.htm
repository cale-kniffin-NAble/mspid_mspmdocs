SLA adjustments based upon a project start/end date

Last Modified

Thu Mar 09 19:20 GMT 2017

Description

Partner Request :

I started a project for a customer but it just lists it into a field. Obviously projects should have a different SLA than what we expect on a day to day. Maybe we need to have adjustments for the SLA based upon a project start/end date with milestones available to benchmark the progress.

Environment

Solution

Steps:
1- Account Settings > SLA Policies > Create a new SLA Policy Profile (Name it Project SLA)
2- Define the number of hours in the SLA Project Profile according to agreement between the partner and his customer
3- Customers > Choose the Customer that the partner has a project with and need use Project SLA Profile > Service Items > Project Service Item > Select the Project SLA Profile
