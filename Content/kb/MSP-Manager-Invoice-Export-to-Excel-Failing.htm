Invoice Export to Excel Failing

Last Modified

Mon Jan 28 20:46 GMT 2019

Description

  • Exporting an invoice to Excel via CSV file produces issues recognizing UTF-8 encoding
  • CSV file opens and displays correctly in text editor

Environment

Solution

  • This is a known issue with Excel not correctly recognizing CSV files encoded with UTF-8
  • If attempting to open a CSV file in Excel in Europe, see this article relating to how Excel separates variables differently by region
  • Workaround 1
    1. Download the CSV file as normal in
    2. Open Excel
    3. Import the CSV file using Data > From Text
    4. Change file type to Delimited
    5. Change file origin to 65001: Unicode (UTF-8)
    6. Click Next
    7. Change Delimiters to Comma (deselect other delimiters)
  • Workaround 2
    1. Download the CSV file as normal in
    2. Open the exported file in an advanced text editor (Notepad++)
    3. Click Encoding menu
    4. Click Convert to UTF-8 BOM
    5. Click Save
    6. Open file in Excel
