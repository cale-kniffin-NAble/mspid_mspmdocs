Tickets created from Workflow rules are assigned to the correct User, however the Status is still "New" and can be viewed using Unassigned Tickets filter

Last Modified

Thu Jun 18 01:24 GMT 2020

Description

  • Tickets assigned to a User via Workflow rules show up as Unassigned ticket.

Environment

Solution

  • Implicit updates to tickets would no longer be applied by workflow. To make this work, Set Property action should be added to the Workflow with the following configuration
    • Select Property = Ticket Status
    • Set Value = Assigned
