Ticket responses create new tickets in

Last Modified

Thu Mar 12 08:06 GMT 2020

Description

  • Ticket responses in are creating new tickets or ticket requests instead of being added as a response to the existing ticket

Environment

Solution

  • Make sure the reference ID is not being deleted in the response
  • The reference ID is a white text (unless highlighted it looks invisible) at the bottom of the email sent to the ticketing mailbox.
  • When you reply to an email that's received by the ticketing mailbox the reference ID is checked against tickets that exist, and add your reply as a response to the ticket assigned that reference ID instead of creating a new ticket.
  • If that reference ID is removed (like when a technician responds to an email and removes the bottom of the original email) when the response is received by the ticketing mailbox and there is no reference ID to check against, a new ticket is created.
