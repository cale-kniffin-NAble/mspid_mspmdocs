Tickets not being auto created or routed properly

Last Modified

Tue May 14 14:20 GMT 2019

Description

  • Sometimes emails for new tickets show up in ticket requests instead of showing up as actual tickets

Environment

Solution

  • This is most commonly resolved by one of the two following solutions:
    • No default service item specified
    • Every customer must have a default service item
    • This has to be set up for each customer one by one manually
    • Everything for automation depends upon having a default service item
    • Set a default service item
    • Domain needs to be specified
    • It should be their domain not a generic domain (not yahoo or google)
  • Note: emails from that domain would be routed to that customer
