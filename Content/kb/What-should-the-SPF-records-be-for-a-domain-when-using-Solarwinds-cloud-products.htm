What should the SPF records be for a domain, when using cloud products?

Last Modified

Thu Dec 03 02:29 GMT 2020

Description

  • When using products, which send notification emails with a configurable from address, what changes should be made to the sending domain's SPF record?

Environment

  • Service Desk

Solution

  • Backup - include:amazonses.com
  • ServiceDesk - include:mailgun.org
  • - include:spf.antispamcloud.com
  • RMM:
    • The records are dashboard region specific:
      • Australia - a:au.mailers.system-monitor.com
      • Germany - a:de.mailers.system-monitor.com
      • Europe, France1, France (FR), Ireland, Poland - a:eu.mailers.system-monitor.com
      • Hong Kong - a:hk.mailers.system-monitor.com
      • UK - a:uk.mailers.system-monitor.com
      • US - a:us.mailers.system-monitor.com
  • Mail Assure - include:spf.mtaroutes.com
NOTE: Each domain should only have one SPF Entry. If you have more than one entry, recipient servers will decline both. As a result, it will cause your emails to fail an SPF check.
