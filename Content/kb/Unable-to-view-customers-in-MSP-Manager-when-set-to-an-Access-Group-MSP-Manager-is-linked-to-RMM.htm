Unable to view customers in when set to an Access Group - is linked to RMM

Last Modified

Thu Apr 25 16:27 GMT 2019

Description

  • Unable to view customers in when set to an Access Group - is linked to RMM
  • These customers are only in and are not shown in RMM
  • The Access group was created in RMM

Environment

  • RMM Integrated
  • Access Groups

Solution

  • This has happened because the Access Group that was created from the RMM Client Group doesn't know about the Standalone Customers in
  • To resolve this:
    1. Have a user with Admin permissions log in to
    2. Navigate to Settings > User Management > Access Groups
    3. Expand the specific Access Group and click on the Edit button
    4. Click on the Customers dialog box and select the Standalone customer(s) to be added to the Access Group
    5. Click Save Access Group
    6. Have the original user log out and back in
