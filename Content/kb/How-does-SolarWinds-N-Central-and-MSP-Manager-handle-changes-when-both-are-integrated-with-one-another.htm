How does and handle changes when both are integrated with one another?

Last Modified

Mon Feb 04 20:36 GMT 2019

Description

How does and handle changes when both are integrated with one another?

Environment

Solution

  • By design, there is no "sync" mechanism in or that detects and manages changes on both
  • If a change is made like changing the customer name in (or vice-versa), the mapped customer name in will not be changed to the updated one
  • Customer name in will not change to match, it needs to be manually updated
