SLA notifications are being sent for completed tickets stating "Ticket has not been In Progress within X hours."

Last Modified

Mon Feb 24 14:48 GMT 2020

Description

  • Ticket resolved and marked complete, but SLA notifications are still being sent stating:
    • "Ticket has not been In Progress within X hours."
    • "As per the SLA, this ticket must be In Progress within X hours."
  • SLA policy is configured to require tickets to be marked in progress within X hours.
  • Ticket never was marked as "In Progress," but was assigned and closed immediately.

Environment

    • SLA Policies

Solution

  1. This issue was resolved by Resolved by BUG 35516 / NCI-15323 on Feb 4, 2020. Should the issue reoccur, the workaround is below, but it is recommended to raise a case to Support, referencing this article.
  • SLA policy configuration requires tickets to be marked "In Progress" before SLA will complete for tickets
  • Mark tickets as "In Progress" while working on tickets, then close ticket after making a note on the ticket, or disable requirement in SLA policy.
    • To mark ticket as "In Progress":
      1. Select Helpdesk
      2. Select ticket
      3. Select Status dropdown and select In Progress
      4. Select Save
    • To disable requirement for tickets to be marked "In Progress":
      1. Select Settings
      2. Select Company Settings
      3. Select SLA Policies
      4. Select pencil icon to edit SLA policy
      5. Select Default Setting
      6. Uncheck Ticket must be marked in progress within X hours
      7. Select Save SLA Policy
